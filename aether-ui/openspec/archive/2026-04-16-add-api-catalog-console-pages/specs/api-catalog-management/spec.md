## ADDED Requirements

### Requirement: 工作台必须通过认证保护管理操作
API Catalog 工作台必须仅对已认证的控制台会话开放访问。

#### Scenario: 匿名用户尝试打开工作台
- **WHEN** 未认证用户导航到工作台路由
- **THEN** 路由守卫在任何管理请求发出前，将用户重定向到控制台登录路由

### Requirement: 工作台必须通过生命周期契约管理分类
工作台必须使用现有分类管理 API 提供分类查询与生命周期操作。

#### Scenario: 运营人员加载分类管理
- **WHEN** 已认证运营人员打开工作台
- **THEN** 页面使用页码与分页大小参数调用 `GET /api/v1/categories`
- **THEN** 页面渲染返回的分类编码、名称、状态与时间戳

#### Scenario: 运营人员变更分类生命周期
- **WHEN** 运营人员创建、重命名、启用或停用某个分类
- **THEN** 页面调用对应的分类契约
- **THEN** 请求成功后，工作台中的分类状态同步更新

### Requirement: 工作台必须通过编码驱动的流程管理资产
由于没有资产列表契约，工作台必须通过草稿创建和显式 `apiCode` 查询来管理资产，而不是展示伪造的服务端资产表。

#### Scenario: 运营人员创建新的资产草稿
- **WHEN** 运营人员提交包含 `apiCode` 与 `assetType` 的资产注册表单
- **THEN** 页面调用 `POST /api/v1/assets`
- **THEN** 返回的资产成为工作台中当前可编辑的资产

#### Scenario: 运营人员加载已有资产
- **WHEN** 运营人员输入 `apiCode` 并发起资产查询
- **THEN** 页面调用 `GET /api/v1/assets/{apiCode}`
- **THEN** 返回的资产成为工作台中当前可编辑的资产

#### Scenario: 运营人员修订资产或变更资产状态
- **WHEN** 运营人员更新当前资产、启用它、停用它，或附加 AI 能力档案
- **THEN** 页面调用对应的资产管理契约
- **THEN** 工作台根据成功响应刷新当前资产快照

### Requirement: 工作台仅对 AI 资产暴露 AI 档案编辑能力
工作台必须仅在当前资产类型为 `AI_API` 时渲染 AI 能力档案相关控件。

#### Scenario: 当前资产为 AI 类型
- **WHEN** 已加载资产满足 `assetType = AI_API`
- **THEN** 工作台渲染提供商、模型、流式支持与能力标签的编辑表单

#### Scenario: 当前资产为标准类型
- **WHEN** 已加载资产满足 `assetType = STANDARD_API`
- **THEN** 工作台隐藏 AI 能力档案表单

### Requirement: 工作台必须标识非权威的本地快捷入口
工作台可以为了操作便利在本地保留最近打开的资产，但不得将该列表标注为完整的服务端资产清单。

#### Scenario: 运营人员再次访问最近资产
- **WHEN** 运营人员在当前浏览器会话中已经成功创建或加载过资产
- **THEN** 工作台可以在“最近打开”区域中渲染这些资产
- **THEN** 该区域文案必须明确说明它只是本地快捷入口，而不是完整资产列表
