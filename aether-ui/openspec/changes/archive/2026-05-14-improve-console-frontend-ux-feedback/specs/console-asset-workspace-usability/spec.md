## ADDED Requirements

### Requirement: 资产新建与编辑 MUST 使用聚焦式浮层流程
控制台资产工作台 MUST 允许用户在聚焦式模态框或抽屉中创建资产草稿和编辑已有资产，并保持资产列表上下文可见或可恢复。

#### Scenario: 用户创建资产草稿
- **WHEN** 已登录用户在资产工作台中开始创建新的 API 资产
- **THEN** 系统 MUST 打开聚焦式创建流程，而不是要求用户跳转到页面下方填写表单
- **THEN** 用户 MUST 能提交或取消草稿创建，且不丢失当前列表筛选条件

#### Scenario: 用户编辑已有资产
- **WHEN** 已登录用户从资产列表选择编辑某个资产
- **THEN** 系统 MUST 打开编辑模态框或抽屉，并填充该资产当前可编辑字段
- **THEN** 编辑流程关闭后，资产列表 MUST 仍作为用户的导航上下文保留

### Requirement: 绑定 AI 能力配置 MUST 保留资产配置状态
控制台资产工作台 MUST 在用户绑定或更新 AI 能力配置时保留已有资产配置字段。

#### Scenario: 用户编辑基础配置后绑定 AI 能力
- **WHEN** 用户已填写或加载资产基础配置，并继续绑定 AI 能力字段
- **THEN** 系统 MUST NOT 清空展示名称、分类、上游地址、请求方法、鉴权方式、请求模板、请求示例、响应示例等无关资产配置字段
- **THEN** 系统 MUST 在当前编辑流程中反映已更新的 AI 能力字段

#### Scenario: AI 能力更新失败
- **WHEN** AI 能力绑定请求失败
- **THEN** 系统 MUST 保留用户尚未保存的基础配置表单值
- **THEN** 系统 MUST 展示国际化的 AI 能力操作错误状态

### Requirement: 资产表单 MUST 解释字段用途与契约边界
控制台资产工作台 MUST 为含义不直观的资产配置字段提供简洁说明。

#### Scenario: 用户查看请求模板和示例字段
- **WHEN** 用户编辑请求模板、请求示例或响应示例字段
- **THEN** 系统 MUST 展示说明，区分“模板结构”和“具体示例载荷”的用途
- **THEN** 说明 MUST 使用国际化文案，并位于相关字段附近

#### Scenario: 用户查看上游和鉴权字段
- **WHEN** 用户编辑上游地址、请求方法、鉴权方式或鉴权配置字段
- **THEN** 系统 MUST 说明这些字段用于配置资产所有者的上游目标
- **THEN** 系统 MUST 明确这些字段不同于公开的平台统一调用地址

### Requirement: 资产列表行 MUST 展示可扫读元信息与操作
控制台资产工作台 MUST 在资产列表行中展示足够的元信息和语义化控件，使用户无需打开每个详情也能扫描、选择和操作资产。

#### Scenario: 用户浏览资产列表
- **WHEN** 资产列表存在资产
- **THEN** 每一行 MUST 展示资产名称或编码、资产类型、生命周期状态，以及请求方法、分类、更新时间、发布时间等可用元信息
- **THEN** 状态和类型 MUST 使用只读 Badge/Tag 样式，而不是操作按钮样式

#### Scenario: 用户操作资产行
- **WHEN** 资产行支持编辑、上架、下架或删除等操作
- **THEN** 每个操作 MUST 与只读状态元信息在视觉上清楚区分
- **THEN** 破坏性操作 MUST 在改变资产状态前要求用户确认

### Requirement: 上架流程 MUST 显示分类依赖
控制台资产工作台 MUST 在发布或配置资产需要分类相关值时，让分类/类型依赖对用户可见。

#### Scenario: 用户配置需要分类信息的资产
- **WHEN** 分类信息对资产发布是必填或推荐项
- **THEN** 编辑流程 MUST 在发布控件附近展示分类字段和说明
- **THEN** 系统 MUST NOT 将该依赖隐藏在不可访问的导航项之后

#### Scenario: 当前 UI 不可管理分类
- **WHEN** 用户需要创建或管理当前屏幕无法处理的分类
- **THEN** 系统 MUST 在用户尝试发布前提供清晰说明或可见的分类管理入口
