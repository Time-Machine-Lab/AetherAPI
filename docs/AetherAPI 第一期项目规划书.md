# AetherAPI 第一期项目规划书

## 1. 本期目标

本期开发周期为一周，目标不是做完整平台，而是完成 Aether API Hub 的第一阶段可用闭环。

本期产品主题明确为：

`AI-ready API Hub MVP`

本期必须验证成功的主链路是：

1. 用户可以在平台发布自己的 API 资产
2. 用户可以在 API 市场浏览已发布资产
3. 用户可以创建 API Key 并通过统一入口调用资产
4. 用户可以在开发者控制台查看自己的调用日志

## 2. 本期范围

### 2.1 必做模块

- API Catalog：用户资产工作台、发布/下架/删除、AI 元数据
- API Marketplace：市场列表、详情、调用入口说明
- Consumer & Auth：控制台登录态、API Key 管理、API 调用鉴权
- Unified Access：统一鉴权、目标解析、直接转发、结果透传
- Observability：最小 API 调用日志记录与查询

### 2.2 产品形态

本期只有两块用户可感知界面：

- API 市场
- 开发者控制台

没有独立管理端。管理员和普通用户都在同一产品中工作，只是承担的操作不同。

## 3. 本期关键语义

### 3.1 资产语义

本期的 API 资产必须按“用户拥有的市场资产”建模：

- 用户可以创建和维护自己的资产
- 用户可以发布自己的资产到市场
- 用户也可以消费市场中其他用户发布的资产

平台负责的是统一管理、统一发现、统一调用链路，不负责代替用户持有资产。

### 3.2 调用语义

当前 Unified Access 不依赖独立网关，而是由 Hub 自行完成最小可用转发。

调用流程为：

1. 用户生成 API Key
2. 调用方携带 API Key 访问 Hub 统一地址
3. Hub 完成 API Key 校验
4. Hub 解析目标已发布资产
5. Hub 向上游发起请求并原样返回结果
6. Hub 记录调用日志

说明：

- 上游响应不使用 TML Result 包装
- 只有平台业务接口使用 TML Result

## 4. 本期交付清单

### 4.1 资产工作台

- 当前用户资产列表
- 当前用户资产详情
- 创建资产
- 编辑资产
- 维护 AI 能力档案
- 发布资产
- 下架资产
- 删除资产

### 4.2 API 市场

- 已发布资产列表
- 已发布资产详情
- 展示发布者基础信息
- 支持普通 API 与 AI API 区分展示

### 4.3 鉴权与控制台

- 控制台登录
- 当前用户 API Key 管理
- 当前用户调用日志查询

### 4.4 统一调用

- 统一入口
- API Key 鉴权
- 目标资产解析
- 上游请求转发
- 上游结果透传

## 5. 本期不做

- 独立管理端
- 订阅/计费/结算体系
- 显式 Consumer 注册流程
- 复杂权限模型
- 重型网关能力
- 完整监控大盘
- MCP Gateway 与 AI 文档自动导入

## 6. 成功标准

本期完成后，Aether API Hub 至少要具备以下能力：

- 用户能发布自己的 API 到市场
- 市场能稳定展示已发布 API
- 调用方能通过 API Key 访问统一调用地址
- Hub 能直接转发并返回上游响应
- 开发者控制台能查看自己的调用日志
- 至少打通普通 API 与 AI API 各一条完整主链路

## 7. 本期结论

本期真正要交付的不是“平台管理 API”，而是“用户在市场中分享和使用 API”的最小闭环。

因此一周内的实现优先级应始终围绕：

- 用户资产发布
- 市场发现
- API Key 调用
- 调用日志回看
